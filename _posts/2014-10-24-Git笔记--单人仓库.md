**如何使用Github创建你的git仓库:**      

**step1:**     
登陆你的github账号，在网页端建立一个仓库。          
**step2:**        
在展示仓库的网页上，找到用于clone的链接。点击复制。     
**step3:**    
在本地terminal中，输入：`git clone xxxxxx(the address you just copied)`     
于是这样你的仓库就被复制到本地了。    
[问题] 为什么一定要从网页端创建？不能本地`git init`一个吗？     
[答疑] 因为你要告诉他push到哪里去啊，`github`只是众多git托管商之一，只有从`github`网站上生成的.git文件才会知道你的代码要托管到`github`上而不是`bitbucket`。     
到此为止，创建完毕。     

#### 密钥创建
ssh-keygen -t rsa -C "youremail@example.com"
然后可在~/.ssh下找到公钥和私钥


#### 仓库的push     
在本地写完代码后，如果想展示在github中，必须要push到远程。步骤：        
{% highlight c %}       
git add .    //表示把当前文件夹下所有文件加入提交范围     
git commit -m "对commit的注释"   //请保持良好的注释习惯     
git push (-u) origin master    //origin是远端仓库的名字，master是分支的名字     
       //-u代表以后如果不注明，可以省略origin master，只输入git push     
{% endhighlight %}       
* 要点1：push前需要设置密钥，把公钥上传到github上。教程：<a href="http://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000/001374385852170d9c7adf13c30429b9660d0eb689dd43a000" style="color: blue;">廖雪峰的教程</a>     
* 要点2：本地的git中邮箱设置一定要于github上的相同，否则提交不计入commit图。参考我的日志：<a href="http://rulerstorm.github.io/git/2014/10/16/关于github不记录contribuions.html" style="color: blue;">关于github不记录contribuions</a>     


#### 分支的使用 
_
##### 什么情况要用分支？     
你不想改变主线（master），却想保存当前修改。在分支上commit多次，代码完整后，可以再与主线合并。     
##### Cheat Sheet     
{% highlight c %}      
git branch                 //查看分支     
git branch branch_name     //创建分支     
git checkout branch_name   //切换至分支     
git checkout -b branch_name  //先创建，再切换     
git merge branch_name      //把这个分支合并到当前分支     
git branch -d branch_name  //删除分支     

//版本回退     
git log			//查看各个commit的注释和hash_number     
git checkout hash_number   //版本回退     
git reflog		//查看所有输入命令时的版本号，用来以防错误回退     
//注意，回滚操作也是基于增量修改的，千万不要手动修改文件，否则就麻烦了！     
{% endhighlight %}       
[注意]小心合并后的冲突，关于冲突，以后再研究。
